# 修复播放器长时间播放卡顿问题 - 进展文档

**日期**: 2026-03-06  
**版本**: v1.7.0 → v1.7.1

## 问题描述
播放器在长时间播放过程中音乐声会一卡一卡，必须手动点击暂停后再播放才能恢复。

## 根因分析

| 问题 | 严重度 | 文件 |
|------|--------|------|
| `initPlaybackGuard` 中用 `timeSincePlay > 1000` 判断用户暂停，导致自动恢复机制失效 | **高** | `preload.js` |
| `blobMap` 只增不减，长时间运行内存泄漏 | 中 | `preload.js` |
| 播放状态轮询频率过高（1秒）增加 GC 压力 | 低 | `preload.js` |
| 心跳检测 60 秒阈值过短，可能误判重载页面 | 中 | `main.js` |

## 修复内容

### preload.js
1. **重写播放保持机制**：使用 `userInitiatedPause` 标志替代不可靠的时间判断，仅当用户实际点击暂停按钮或托盘控制时才标记为用户暂停
2. **增加媒体事件监听**：新增 `waiting`、`stalled`、`error` 事件处理，主动检测和恢复缓冲/网络问题
3. **添加 `visibilitychange` 拦截**：防止窗口切换导致网页播放器自动暂停
4. **修复 blobMap 内存泄漏**：重写 `revokeObjectURL` 同步清理，并添加定期清理（每5分钟）
5. **降低轮询频率**：`checkPlayState` 从 1 秒改为 3 秒

### main.js
1. **心跳阈值增至120秒**：减少因短暂忙碌导致的误判页面重载
2. **添加音频保活**：每10分钟确保音频未被静音、电源保持锁有效
3. **托盘控制联动**：暂停/播放操作发送 `toggle-play-guard` 通知

## 验证
- ✅ `preload.js` 和 `main.js` 语法检查通过
